# Scanner Ghosts -- The Security Scanner Archive

> Preserving the history of early web and network security scanning tools before it's gone.

---

## What This Is

This is a community archive of **web security scanning tools**, with a focus on tools released **before 2005**. Early infosec tooling is gone or rapidly disappearing. The goal of this project is to document and preserve these tools — their code, their documentation, and their historical context — while that's still possible.

This is not a tool *for* scanning. It is a library *about* scanners.

---

## Why It Matters

The period from roughly 1993–2005 was formative for web security. Tools like Whisker, cgichk, and early Nikto defined how we thought about web vulnerability scanning. Many of them were only distributed on message boards, FTP servers, BBSs, or see-privately, and were never even posted on personal sites, let alone archived. Even those personal sites are gone, and many of them were sadly never captured by The Wayback Machine.

In some cases, these tools exist only in people's memories or in forgotten corner of an old hard drive.

This project exists to change that.

---

## Research

Reconstructing this history is hard, and much of it is still unsettled. Dates
are fuzzy, attribution is by handle, and the same code was repackaged under
different names. A tool's *internal* code timestamp, the date it was *posted*
to a public archive, and the date it was actually *written* can all differ by
months. We try to separate what is **documented** from what is **remembered**,
and we flag anything unconfirmed as exactly that.

**What counts as evidence**, strongest first:

- Original source code, with intact headers, comments, and internal timestamps
- Dated mailing-list / newsgroup posts (Bugtraq, EFnet `#hack` logs, ezines such as HWA)
- Archived distribution pages (Packet Storm IDs, Wayback captures with timestamps)
- Firsthand accounts from authors or contemporaries — useful context, but memory is not proof

### Open question: what was the first public CGI/web scanner?

This is unresolved. The current state of the evidence:

- **Bronc Buster (LoU) scanner (~Nov 1998 / Jan 1999)** — the earliest candidate.
  Packet Storm [id/12206](https://packetstormsecurity.com/files/12206/) is dated January 1999 and described as "older." A
  second file, [id/12316](https://packetstormsecurity.com/files/12316/), carries a code comment claiming *"coded by Bronc
  Buster of LoU - Nov 1998"* — which would be the earliest dated artifact in the
  set **if verified**. Currently unconfirmed; we need the source or a dated post
  to prove it. One contemporary recalls using it in 1999 as an early influence on
  web hacking, in contrast to the memory-corruption/shellcode focus of the era.
- **CKS & Fdisk / cgichk / CGI Checker lineage (May–Aug 1999)** — `cgichk
  v1.34.2` (su1d sh3ll, posted 1999-08-17, Packet Storm [id/11875](https://packetstormsecurity.com/files/11875/)) does 60 CGI
  checks, carries an internal code timestamp of `13.05.99`, and credits original
  authors *"[CKS & Fdisk]."* This is the earliest *precisely dated* artifact in
  this branch, but it is later than the Bronc Buster candidate. `CGI Checker
  v1.35` (65 checks, HWA ezine #19) and `cgichk v2.6.0` are later iterations of
  the same codebase. Notably, the Bronc Buster [id/12206](https://packetstormsecurity.com/files/12206/) and this cgichk file
  share a posting date — near-simultaneous public release, even if the
  underlying code dates differ.
- **`|MMX|` exe (1999-06-23)** — sits between the cgichk `13.05.99` internal date
  and the 1999-08-17 posting, within the same mid-1999 cluster. Labeled "version
  2" of an earlier tool that has not been found.

Before any public scanner existed, CGI vulnerabilities were reportedly traded
informally on EFnet's `#hack`. The true "first" *knowledge* of these bugs likely
predates any tool — possibly recorded in a t-file that has not yet surfaced.

### What we're looking for

- Source for the **Bronc Buster / LoU** scanner ([id/12316](https://packetstormsecurity.com/files/12316/) any earlier version) to confirm the Nov 1998 claim
- The unfound **"version 1"** that `|MMX|` was derived from
- Dated **mailing-list or newsgroup posts**, EFnet `#hack` logs, or messaging discussing CGI vulns/scanners pre-2000
- Any **CKS & Fdisk** originals predating the su1d sh3ll repackaging

Have a file, a post, or a firsthand account? Open an issue — even a lead with no
write-up helps.

## Contributing

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full guide.

**Quick version:**

1. Copy `schema/tool-template.md` to `tools/web/[toolname]/README.md`
2. Fill out `meta.yml` (copy `schema/meta-template.yml`) and the prose — partial entries are welcome
3. If you have source code, add the original archive to `src/` (e.g. `src/whisker-1.4.tar.gz`) — preserve it exactly, do not extract or modify
4. Run `python3 scripts/archive.py generate` to build the tables, then `check` to validate
5. Open a pull request

**Know of a tool that isn't listed?** Open an issue using the [Add a Tool](../../issues/new?template=add-a-tool.md) template — you don't need to do the write-up yourself.

---

## Principles

- **Preserve exactly.** Source code goes in unchanged. Don't fix bugs, don't modernize syntax.
- **Cite everything.** Note where every file came from and when it was retrieved.
- **Partial is better than nothing.** An incomplete entry is more useful than no entry.
- **Context matters.** A tool without historical context is just code. Explain what it was for and why it existed.
- **Respect authorship.** Credit original authors clearly. If a license is unknown, say so; don't assume.
